# Security

## Authentication Flow

### Registration

1. User submits username + password to `POST /api/auth/register`.
2. Server checks `ALLOW_REGISTRATION` env var. If `false`, return `403 Forbidden`.
3. Username validated: alphanumeric + underscore, 3-32 characters.
4. Password validated: minimum 8 characters, must contain uppercase, lowercase, and number.
5. Password hashed using **Argon2id** with:
   - **Per-user random salt** (16 bytes, generated by `crypto.randomBytes`).
   - **Application-level pepper** (from `AUTH_PEPPER` env var, concatenated before hashing).
6. User record inserted into database with UUID v4.
7. Session cookie created and returned.

### Login

1. User submits username + password to `POST /api/auth/login`.
2. User fetched by username from database.
3. If user exists, password verified:
   - Pepper from `AUTH_PEPPER` env var concatenated with submitted password.
   - Argon2id verify against stored hash (salt embedded in hash).
4. On success: new session created with cryptographically random token (32 bytes).
5. Session token hashed (SHA-256) before storage in `sessions` table.
6. Session cookie set with flags: `HttpOnly`, `Secure`, `SameSite=Lax`, `Path=/`.
7. Session expires after 30 days of inactivity.

### Session Validation

1. Session cookie read from request.
2. Cookie value hashed (SHA-256) and looked up in `sessions` table.
3. If found and `expires_at > now`, session is valid.
4. User ID extracted from session record.
5. Sliding expiration: `expires_at` extended on each valid request.

### Logout

1. Session cookie read from request.
2. Session record deleted from database.
3. Cookie cleared with `Max-Age=0`.

## Password Hashing: Argon2id + Salt + Pepper

### Why Three Layers?

| Layer | Purpose | Storage |
|-------|---------|---------|
| **Argon2id** | Memory-hard algorithm, resistant to GPU/ASIC attacks | Algorithm parameters |
| **Salt** | Prevents rainbow table attacks, ensures unique hashes per password | Embedded in hash string |
| **Pepper** | Prevents database-only breaches from being useful | `AUTH_PEPPER` env var (NOT in DB) |

### Implementation

```typescript
import { hash, verify } from 'argon2';
import crypto from 'crypto';

const PEPPER = process.env.AUTH_PEPPER; // Required env var

async function hashPassword(password: string): Promise<string> {
  const salt = crypto.randomBytes(16).toString('hex');
  const pepperedPassword = `${password}${PEPPER}${salt}`;
  
  return hash(pepperedPassword, {
    type: 2, // Argon2id
    memoryCost: 65536, // 64 MB
    timeCost: 3,
    parallelism: 1,
    salt: Buffer.from(salt, 'hex'),
  });
}

async function verifyPassword(password: string, hash: string): Promise<boolean> {
  // Salt is extracted from the stored hash by argon2 library
  const pepperedPassword = `${password}${PEPPER}${extractSalt(hash)}`;
  return verify(hash, pepperedPassword);
}
```

### Argon2id Parameters

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Type | Argon2id (2) | Hybrid: resistant to both side-channel and GPU attacks |
| Memory Cost | 65536 (64 MB) | High memory requirement defeats GPU cracking |
| Time Cost | 3 iterations | Balance between security and login speed (~200ms) |
| Parallelism | 1 | Single-threaded for consistent timing |
| Salt Length | 16 bytes | Industry standard |
| Hash Length | 32 bytes | Industry standard |

## Visibility & Access Control

### PRIVATE Snippets

- **Query filter:** `WHERE author_id = ? AND visibility = 'PRIVATE'`
- **API guard:** Middleware validates session, extracts user ID.
- **Error:** `404 Not Found` (not `403` - do not leak existence).

### SHARED Snippets

- **Access method:** `/snippets/[id]?token=[shareToken]`
- **Query:** `WHERE id = ? AND share_token = ? AND visibility = 'SHARED'`
- **If token missing or invalid:** `404 Not Found`.
- **No listing:** SHARED snippets never appear in any list endpoint.
- **No auth required:** Token is the sole access mechanism.

### PUBLIC Snippets

- **Query:** `WHERE visibility = 'PUBLIC'` (no auth required).
- **Listed on:** Public homepage explorer.
- **Detail view:** Anyone can view, but only author can edit/delete.

### Authorization Matrix

| Action | PRIVATE | SHARED | PUBLIC |
|--------|---------|--------|--------|
| View (author) | Yes | Yes | Yes |
| View (anyone) | No | With token | Yes |
| List (author) | Yes | Yes | Yes |
| List (anyone) | No | No | Yes |
| Edit | Author only | Author only | Author only |
| Delete | Author only | Author only | Author only |
| Share | Author only | Author only | Author only |

## HTTP Security Headers (Caddy)

| Header | Value | Purpose |
|--------|-------|---------|
| `Content-Security-Policy` | `default-src 'self'; script-src 'self' 'nonce-{random}'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self'; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'` | Prevents XSS, clickjacking, data exfiltration |
| `Strict-Transport-Security` | `max-age=63072000; includeSubDomains; preload` | Forces HTTPS for 2 years |
| `X-Content-Type-Options` | `nosniff` | Prevents MIME-type sniffing |
| `X-Frame-Options` | `DENY` | Prevents clickjacking |
| `X-XSS-Protection` | `0` | Disabled (CSP is superior) |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | Limits referrer leakage |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` | Restricts browser features |

## SQL Injection Prevention

- **Drizzle ORM** parameterized queries exclusively.
- No raw SQL string concatenation with user input.
- All input validated with Zod schemas before reaching database layer.
- SQLite's `better-sqlite3` prepared statements under the hood.

## Session Security

- Session tokens: 32 bytes from `crypto.randomBytes()`.
- Stored as SHA-256 hash (plaintext token never stored).
- Cookie flags: `HttpOnly`, `Secure`, `SameSite=Lax`.
- Session rotation on privilege changes.
- Automatic cleanup of expired sessions (cron or on-startup).

## Environment Variables (Secrets)

| Variable | Required | Description |
|----------|----------|-------------|
| `AUTH_PEPPER` | Yes | Application-level pepper for password hashing |
| `SESSION_SECRET` | Yes | Secret for session token signing/encryption |
| `DATABASE_URL` | Yes | Path to SQLite database file |
| `ALLOW_REGISTRATION` | Yes | `true` or `false` - controls registration endpoint |
| `NODE_ENV` | Yes | `production` or `development` |

**NEVER commit `.env` files to version control.**
